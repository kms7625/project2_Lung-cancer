# LIDC-IDRI 데이터셋 구조와 모델 훈련 파이프라인

---

## 1. 원본 데이터가 뭔가요?

LIDC-IDRI는 **1,010명 환자의 폐 CT 스캔** 데이터셋입니다.  
4명의 방사선과 전문의가 각 CT에서 결절(nodule)을 찾아 폴리곤으로 윤곽을 그리고, **악성도 점수(1~5)**를 매겼습니다.

```
원본 데이터 구성
├── DICOM 파일  → CT 영상 (환자 1명당 수백 장의 단면 이미지)
└── XML 파일    → 결절 위치(폴리곤 좌표) + 악성도 점수
```

---

## 2. 전처리 후 파일 구조

`resize_ct.py`가 원본 DICOM + XML을 처리해서 아래처럼 저장합니다.

```
/home/kms/resnet_project/lidc-idri/
│
├── slices/                          ← 전처리된 CT 단면 이미지
│   ├── LIDC-IDRI-0001/
│   │   ├── slice_048_4.npy          ← 48번째 단면, 악성도 4
│   │   ├── slice_052_3.npy          ← 52번째 단면, 악성도 3
│   │   └── ...
│   ├── LIDC-IDRI-0002/
│   │   └── ...
│   └── ... (총 889명)
│
├── nodule_malignancy_scores.json    ← 결절 위치(폴리곤 좌표) 정보
└── labels.csv                       ← 훈련에 사용하는 최종 표
```

## 3. NPY 파일이 뭔가요?

`.npy` 파일 하나 = **CT 단면 이미지 1장** (300 × 300 픽셀)

```
파일명: slice_048_4.npy
           │      │
           │      └── 악성도 점수 (1~5)
           └───────── 슬라이스 번호 (CT를 z축으로 자른 순서)
```

파일 안에는 **HU(Hounsfield Unit)** 값이 담겨 있습니다.  
HU는 조직 밀도를 나타내는 숫자입니다.

```
HU 값    의미
-1000   공기 (폐 안 빈 공간)
  -50   지방
    0   물
  +40   혈액
 +700   뼈
```

---

## 4. JSON 파일이 뭔가요?

`nodule_malignancy_scores.json`은 **결절의 정확한 위치(폴리곤 좌표)**를 담고 있습니다.

```json
{
  "LIDC-IDRI-0001": {
    "slice_48": {
      "reader_0": [
        {
          "noduleID": "Nodule 001",
          "score": 4,
          "polygon": [[60, 170], [65, 172], [68, 178], ...]
        }
      ],
      "reader_1": [...],
      "reader_2": [...],
      "reader_3": [...]
    }
  }
}
```

- **reader_0~3**: 4명의 방사선과 의사가 각자 그린 결절 윤곽
- **polygon**: 결절 테두리의 픽셀 좌표 목록
- **score**: 악성도 점수 (1=양성, 5=악성)

이 좌표들의 평균을 내면 **결절의 중심 위치(centroid)** 를 알 수 있습니다.

```
centroid_x = 모든 폴리곤 x좌표의 평균  (좌우 위치, 0~300)
centroid_y = 모든 폴리곤 y좌표의 평균  (상하 위치, 0~300)
```

## 5. prepare_csv.py가 하는 일

`prepare_csv.py`는 위의 파일들을 읽어서 **훈련용 표(labels.csv)** 를 만듭니다.

### Step 1 — NPY 파일 전체 스캔

```
slices/ 폴더를 재귀적으로 탐색
→ slice_048_4.npy 발견
→ 환자ID = LIDC-IDRI-0001 (폴더명)
→ 슬라이스번호 = 48 (파일명 두 번째 숫자)
→ 악성도 = 4 (파일명 마지막 숫자)
→ 라벨 = 1 (악성, score 4~5)
```

### Step 2 — 라벨 결정 (전략 A)

```
score 1, 2, 3  →  label = 0 (양성)
score 4, 5     →  label = 1 (악성)
```

### Step 3 — 환자 단위 분할

```
889명 환자를 섞은 뒤 비율에 맞게 나눔

Train : 623명 (70%)  →  5,530개 슬라이스
Val   : 133명 (15%)  →  1,111개 슬라이스
Test  : 133명 (15%)  →  1,208개 슬라이스
```

> **왜 환자 단위로 나누나요?**  
> 같은 환자의 슬라이스를 train/test에 동시에 넣으면 모델이 "이 환자는 악성이었지"를  
> 기억해버려서 성능이 뻥튀기됩니다. 이를 **Data Leakage** 라고 합니다.

### Step 4 — JSON에서 centroid 추출

```
JSON에서 LIDC-IDRI-0001 / slice_48 찾기
→ 4명 의사의 모든 폴리곤 좌표 수집
→ x 좌표 평균 = cx = 60.84
→ y 좌표 평균 = cy = 177.11
```

### Step 5 — CSV 저장

| patient_id | image_path | label | split | cx | cy |
|---|---|---|---|---|---|
| LIDC-IDRI-0001 | .../slice_048_4.npy | 1 | train | 60.84 | 177.11 |
| LIDC-IDRI-0001 | .../slice_052_3.npy | 0 | train | 62.10 | 180.30 |
| ... | | | | | |

## 6. 훈련 시 이미지를 어떻게 처리하나요?

`train_baseline.py`의 `LIDCDataset`이 CSV 한 행씩 읽으면서 아래 과정을 수행합니다.

```
① NPY 로드
   slice_048_4.npy  →  (300, 300) 배열, HU 값 (정수)

② HU 윈도잉 (정규화)
   clip(-1000 ~ 400) 후 → (값 + 1000) / 1400
   → 픽셀값이 0.0 ~ 1.0 사이 실수로 변환
   (폐 결절이 잘 보이는 범위만 남기고 나머지는 잘라냄)

③ ROI 크롭 (결절 중심 기준)
   cx=60, cy=177 기준으로 128×128 영역 잘라냄
   → 결절이 이미지 중앙에 오도록 집중
   (cx/cy가 없으면 이미지 중심 150,150으로 대체)

④ 3채널 복제
   (128, 128) → (128, 128, 3)
   (흑백 이미지를 RGB처럼 만들어 ImageNet 사전학습 모델에 입력 가능하게)

⑤ Resize + Normalize
   (128, 128) → (224, 224) 리사이즈
   픽셀 평균 0.5, 표준편차 0.5로 정규화

⑥ Augmentation (Train만)
   RandomHorizontalFlip, RandomVerticalFlip, RandomRotation(15°)
   RandomErasing(p=0.2)  ← 일부 영역을 랜덤하게 지워서 과적합 방지
```

## 7. 모델이 뭘 학습하나요?

```
입력: 224×224 CT 단면 이미지 (결절 주변 크롭)
출력: [양성일 확률, 악성일 확률]

예) [0.12, 0.88] → 악성으로 분류
```

EfficientNetV2-S 모델이 이미지의 특징을 추출하고,  
CBAM 어텐션 모듈이 결절 영역에 더 집중하도록 돕습니다.

---

## 8. 전체 흐름 요약

```
DICOM + XML
    ↓  resize_ct.py
슬라이스 NPY 파일 (300×300, HU값)
+ nodule_malignancy_scores.json (결절 위치)
    ↓  prepare_csv.py
labels.csv (환자 단위 분할 + centroid 좌표)
    ↓  train_baseline.py (LIDCDataset)
HU 윈도잉 → ROI 크롭 128×128 → 224×224 리사이즈
    ↓
EfficientNetV2-S + CBAM
    ↓
양성 / 악성 분류
```
